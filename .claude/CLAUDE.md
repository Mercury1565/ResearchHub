# ResearchHub — Claude Instructions

## Project Overview

ResearchHub is a web-based research workspace. Users upload PDFs, annotate them
with highlights and sticky notes, cross-reference documents via deep links, and
query an AI chat scoped to their active project (RAG).

```
research_hub/
├── server/          # Go 1.22 REST API
└── frontend/        # React 18 + Vite 5 SPA
```

## Critical Conventions

### Go

- Module path is `github.com/researchhub/server` — always use this in imports.
- All SQL queries live in `internal/db/queries/*.sql` and are code-generated by
  `sqlc`. Never write raw SQL inline in handlers or services.
- `internal/db/generated/` is machine-generated — never hand-edit it. Re-run
  `sqlc generate` after any `.sql` change.
- Handlers only call services. Services only call the DB layer. Never skip a layer.
- Wrap all errors: `fmt.Errorf("serviceName.FuncName: %w", err)`.
- File uploads must validate size (≤ 25 MB) and MIME type (`application/pdf`)
  before any processing begins.
- Vector search must always be scoped by `project_id` via a JOIN — never run a
  full-table ANN query.
- SSE streaming handlers must call `flusher.Flush()` on every token to meet the
  2.5 s first-token latency target.
- Every handler must have a complete swaggo annotation block. See the Swagger
  section below for the required format. Run `swag init` after any annotation
  change — never hand-edit `docs/`.

### TypeScript / React

- All shared types come from `src/types/index.ts` — never define types inline in
  components or API files.
- All server calls go through React Query hooks in `src/api/` — never fetch
  directly inside a component.
- Annotation coordinates are stored as **0–1 fractions** of page size. Convert to
  pixels only at render time inside the SVG layer.
- The PDF canvas must re-render when `zoom` changes — the `useEffect` must depend
  on `[pdfPage, zoom, devicePixelRatio]`.
- The annotation SVG overlay is `pointer-events-none` on the `<svg>` itself;
  only individual `<rect>` elements receive pointer events.
- `pdfjs-dist` worker path is always `/pdf.worker.min.mjs` (served from `public/`).
  Never point it at a CDN.
- Deep link navigation goes through Zustand + React Router — never `window.location`.

### Styling

- Follow `researchhub-design` skill exactly for all colors, spacing, and components.
- The palette is monochrome-first. The only accent color is `#2383E2` and it is
  used only for interaction feedback (selected state, primary CTA, focus ring).
- No `shadow-md` or larger on any application shell element.
- No `font-bold` (700). Max weight is `font-semibold` (600), used sparingly.
- Transitions are `transition-colors duration-100` only — never animate layout
  properties (width, height, padding).

## Object Storage (MinIO / S3-compatible)

All PDF files are stored in an S3-compatible object store — **MinIO** locally,
swappable for AWS S3, Cloudflare R2, or Backblaze B2 in production with only
an env var change.

**SDK:** `github.com/minio/minio-go/v7` — the MinIO Go client is fully
S3-compatible and works against real AWS S3 without any code changes.

### Storage service interface (`internal/services/storage.go`)

```go
type StorageService interface {
    // Upload stores a PDF and returns its permanent object key.
    Upload(ctx context.Context, key string, r io.Reader, size int64) error

    // PresignedURL returns a time-limited URL for the frontend to fetch the PDF.
    // TTL should be short (e.g. 15 minutes) — never expose permanent URLs.
    PresignedURL(ctx context.Context, key string, ttl time.Duration) (string, error)

    // Delete removes the object. Called when a document is deleted.
    Delete(ctx context.Context, key string) error
}
```

### Object key convention

```
pdfs/{document_id}.pdf
```

Store only the key in `documents.storage_path` — never a full URL.
Generate presigned URLs on demand in the `GET /documents/{id}/file` handler.

### MinIO local setup (Docker)

```bash
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  quay.io/minio/minio server /data --console-address ":9001"
```

Create the bucket once:
```bash
# via MinIO console at http://localhost:9001
# or via mc CLI:
mc alias set local http://localhost:9000 minioadmin minioadmin
mc mb local/researchhub
```

### Switching to production S3

Only env var changes needed — no code changes:
```
S3_ENDPOINT=https://s3.amazonaws.com   # or leave blank for AWS SDK default
S3_ACCESS_KEY_ID=AKIA...
S3_SECRET_ACCESS_KEY=...
S3_BUCKET=researchhub-prod
S3_REGION=eu-west-1
S3_USE_SSL=true
```

### Rules

- Never store raw file paths or local disk paths — always use object keys.
- Never return a permanent object URL — always generate a short-lived presigned URL.
- The `documents.storage_path` column stores the object key only (`pdfs/{id}.pdf`).
- Bucket must have private ACL — no public access.

## LLM Provider System

The active provider is controlled entirely by `LLM_PROVIDER` in `.env`.
No code changes are needed to switch providers — only a config change.

### Provider constants (`config/config.go`)

```go
const (
    ProviderGemini    LLMProvider = "gemini"
    ProviderAnthropic LLMProvider = "anthropic"
    ProviderOpenAI    LLMProvider = "openai"
)
```

`config.ActiveAPIKey()` returns the correct key for whichever provider is set.
Services must use this method — never read a specific key field directly.

### Adding a new provider

1. Add a new `Provider*` constant to `config/config.go`.
2. Add the corresponding `*APIKey` field and env var.
3. Add a case to `ActiveAPIKey()`.
4. Implement the provider in `internal/services/` behind the shared interfaces:
   - `EmbeddingService` — `Embed(ctx, text) ([]float32, error)`
   - `ChatService` — `Stream(ctx, messages, onToken func(string)) error`
5. Wire the new implementation in `cmd/server/main.go` based on `cfg.LLMProvider`.
6. Update `.env.example` with the new key.

### Current provider: Gemini

- Embeddings: `text-embedding-004` (768-dim) — note this differs from the
  default 1536 in the schema; update `vector(768)` in the migration if using
  Gemini embeddings from the start.
- Chat: `gemini-1.5-flash` for cost efficiency, `gemini-1.5-pro` for quality.
- SDK: `google.golang.org/genai` (official Go SDK).

### Planned providers

## Swagger / API Documentation

**Tool:** `swaggo/swag` — generates OpenAPI 2.0 (`swagger.json` / `swagger.yaml`)
from Go annotation comments.
**UI:** `swaggo/http-swagger` serves the Swagger UI at `GET /swagger/*`.

### Setup (already in go.mod)

```
github.com/swaggo/swag
github.com/swaggo/http-swagger
```

Install the CLI once:
```bash
go install github.com/swaggo/swag/cmd/swag@latest
```

### Main entry annotation (`cmd/server/main.go`)

The top-level `@` block lives directly above `package main`. Update the `@host`
and `@BasePath` if the port or prefix ever changes.

```go
// @title           ResearchHub API
// @version         1.0
// @description     REST API for the ResearchHub research workspace.
// @termsOfService  http://swagger.io/terms/

// @contact.name   ResearchHub Support
// @contact.email  support@researchhub.dev

// @license.name  MIT

// @host      localhost:8080
// @BasePath  /api

// @securityDefinitions.apikey BearerAuth
// @in header
// @name Authorization

package main
```

### Route registration (router.go)

Register the Swagger UI handler on the router — outside the `/api` group:

```go
import httpSwagger "github.com/swaggo/http-swagger"
import _ "github.com/researchhub/server/docs"  // blank import triggers init()

// In NewRouter, before the /api group:
r.Get("/swagger/*", httpSwagger.Handler(
    httpSwagger.URL("/swagger/doc.json"),
))
```

### Handler annotation format

Every handler gets an annotation block in the comment directly above the function.
Use this template — fill in all fields, never leave placeholders:

```go
// CreateProject creates a new project.
//
// @Summary      Create project
// @Description  Creates a new project with the given name.
// @Tags         projects
// @Accept       json
// @Produce      json
// @Param        body  body      CreateProjectRequest  true  "Project name"
// @Success      201   {object}  models.Project
// @Failure      400   {object}  ErrorResponse
// @Failure      500   {object}  ErrorResponse
// @Router       /projects [post]
func (h *Handler) CreateProject(w http.ResponseWriter, r *http.Request) {
```

### Request / response types

Define explicit request structs in the same handler file and annotate their fields.
Never use `map[string]any` as a Swagger type — it produces useless docs.

```go
// CreateProjectRequest is the body for POST /projects.
type CreateProjectRequest struct {
    Name string `json:"name" example:"My Research Project"`
}

// ErrorResponse is returned on all 4xx and 5xx responses.
type ErrorResponse struct {
    Error string `json:"error" example:"invalid request body"`
}
```

Place `ErrorResponse` in `internal/api/respond.go` so it can be referenced
across all handler files.

|-----|--------|
| `projects` | Project CRUD |
| `documents` | Document upload and retrieval |

### SSE endpoint (chat)

The `/chat` endpoint streams — annotate it clearly so consumers know not to
expect a standard JSON body:

```go
// StreamChat streams AI responses for a project.
//
// @Summary      Stream AI chat
// @Description  Streams token-by-token AI responses over Server-Sent Events (SSE).
//               Set Accept: text/event-stream. Each event is a JSON token chunk.
//               A final event with data: [DONE] signals end of stream.
// @Tags         chat
// @Accept       json
// @Produce      text/event-stream
// @Param        projectID  path      string           true  "Project UUID"
// @Param        body       body      ChatRequest      true  "User message"
// @Success      200        {string}  string           "SSE stream"
// @Failure      400        {object}  ErrorResponse
// @Router       /projects/{projectID}/chat [post]
```

### Regenerating docs

Run this from inside `server/` any time handler annotations change:

```bash
swag init -g cmd/server/main.go --output docs
```

This overwrites `docs/docs.go`, `docs/swagger.json`, and `docs/swagger.yaml`.
Commit all three. Never edit them by hand.

## What Not to Do

- Do not commit `.env` files.
- Do not hand-edit `internal/db/generated/` — always regenerate with `sqlc`.
- Do not hand-edit `docs/` — always regenerate with `swag init`.
- Do not add a handler without a complete swaggo annotation block.
- Do not store files on local disk — all PDFs go through the S3 storage service.
- Do not store full URLs in the database — store only the object key; generate presigned URLs at request time.
- Do not expose permanent S3 URLs — always use short-lived presigned URLs.
- Do not add new color values or shadows outside the design skill's palette.
- Do not introduce new third-party Go packages without updating `go.mod` and
  documenting the reason here.
- Do not introduce new npm packages without checking if the functionality is
  already available in the existing stack.
- Do not write business logic in HTTP handlers — it belongs in `internal/services/`.
- Do not use `any` in TypeScript — use proper types from `src/types/index.ts`.
